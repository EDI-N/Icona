    <span id="h31232e4764c5e5a6a673749697b8027"></span><h1>How to the Readthedocs<a class="headerlink" href="#how-to-the-readthedocs" title="Permalink to this headline">¶</a></h1>
    <p>The <a href="https://readthedocs.org/" target="_blank">readthedocs.org</a> (RTD) is an awesom place to host your documentation of projects in the Github as well as other repositories.</p>
    <p>This is a simple guideline to help the GGeditor users to set up a 
    basic project of the RTD. I have no idea if this is the best practice, 
    it just works for me.</p>
    <ol class="arabic simple">
    <li>Create a Google Docs document and bind it to the “docs/index.rst” in the Github repository.</li>
    <li>Insert markups of “toctree” table into index.rst.</li>
    <li>Create “docs/conf.py” on Github</li>
    <li>Create “docs/static/theme_overrides.css” on Github</li>
    </ol>
    <p>That’s all. With the above 4 steps, you can go the RTD and create a project for your repository.</p>
    <p>Below, I will show you how to do these steps with the GGeditor.</p>
    <div class="section" id="step-1-binding-to-the-docs-index-rst">
    <span id="h467c3456c435f3c292f45222c3d4910"></span><h2>Step 1: Binding to the docs/index.rst<a class="headerlink" href="#step-1-binding-to-the-docs-index-rst" title="Permalink to this headline">¶</a></h2>
    <p>You can bind the Google Docs document with the GGeditor by menu item 
    “Add-ons/GGeditor/Commit to Github”. Then select your repository, docs 
    folder and the index.rst file in the dialog.</p>
    <p>If the “index.rst” file does not exist in your Github repository, you
     can navigate to the “docs” folder, then click “New File” and input 
    “index.rst”. The GGeditor will create it.</p>
    <p><a class="reference internal" href="https://ggeditor.readthedocs.io/en/latest/_images/how2Readthedocs_1.png"><img alt="IMG1" src="How%20to%20the%20Readthedocs%20%E2%80%94%20GGEditor%20documentation_files/how2Readthedocs_1.png" style="width: 502px; height: 285px;"></a></p>
    <blockquote>
    <div>Or select “New File” to create.</div></blockquote>
    <p><a class="reference internal" href="https://ggeditor.readthedocs.io/en/latest/_images/how2Readthedocs_2.png"><img alt="IMG2" src="How%20to%20the%20Readthedocs%20%E2%80%94%20GGEditor%20documentation_files/how2Readthedocs_2.png" style="width: 357px; height: 180px;"></a></p>
    <p>If even the “docs” folder does not exist yet in your Github 
    repository, you can input “docs/index.rst” as the name in the dialog of 
    the “New File”. Like this:</p>
    <p><a class="reference internal" href="https://ggeditor.readthedocs.io/en/latest/_images/how2Readthedocs_3.png"><img alt="IMG3" src="How%20to%20the%20Readthedocs%20%E2%80%94%20GGEditor%20documentation_files/how2Readthedocs_3.png" style="width: 364px; height: 186px;"></a></p>
    </div>
    <div class="section" id="step-2-insert-toctree-markup-to-the-index-rst">
    <span id="h195ff4c157e501d115f391d4e173b36"></span><h2>Step 2: insert toctree markup to the index.rst<a class="headerlink" href="#step-2-insert-toctree-markup-to-the-index-rst" title="Permalink to this headline">¶</a></h2>
    <p>The index.rst is the homepage of your project in the RTD. You can put
     anything you want. But you should not miss the awesome feature - the 
    sidebar of “table of contents” for your documentation. Just simply do 
    this in the GGeditor markup panel:</p>
    <ol class="arabic simple">
    <li>Insert a “Table of Contents” from the “GGeditor Markup Panel / Markup / Directives”</li>
    </ol>
    <p><a class="reference internal" href="https://ggeditor.readthedocs.io/en/latest/_images/how2Readthedocs_4.png"><img alt="IMG4" src="How%20to%20the%20Readthedocs%20%E2%80%94%20GGEditor%20documentation_files/how2Readthedocs_4.png" style="width: 310px; height: 493px;"></a></p>
    <p>Then, the document would have a “toctree” table like this:</p>
    <p><a class="reference internal" href="https://ggeditor.readthedocs.io/en/latest/_images/how2Readthedocs_5.png"><img alt="IMG5" src="How%20to%20the%20Readthedocs%20%E2%80%94%20GGEditor%20documentation_files/how2Readthedocs_5.png" style="width: 232px; height: 274px;"></a></p>
    <p>You can reference <a href="https://docs.google.com/document/d/13b5dr8TZoTC5IJZeoiDt066b6mwq67yHqcl4TYUFnk0/edit?usp=sharing" target="_blank">the index.rst of the GGeditor</a> for example.</p>
    </div>

    <div class="section" id="step-4-theme-overrides-css">
    <span id="h4a47434f5c5745347cc5f1b4d2d5023"></span><h2>Step 4: theme_overrides.css<a class="headerlink" href="#step-4-theme-overrides-css" title="Permalink to this headline">¶</a></h2>
    <p>You can use the same way to create a new “theme_overrides.css” in the “docs/static” folder. Like this:</p>
    <p><a class="reference internal" href="https://ggeditor.readthedocs.io/en/latest/_images/how2Readthedocs_8.png"><img alt="IMG8" src="How%20to%20the%20Readthedocs%20%E2%80%94%20GGEditor%20documentation_files/how2Readthedocs_8.png" style="width: 626px; height: 149px;"></a></p>
    <p>Below is the content of the theme_overrides.css for you to copy and paste.</p>
    <div class="code highlight-default"><div class="highlight"><pre><span></span>.wy-table-responsive table td, .wy-table-responsive table th {
       white-space: inherit;
    }
    
    .wy-table-responsive table th {
       background-color: #f0f0f0;
    }
    
    .line-block, .docutils.footnote {
        line-height: 24px;
    }
    
    .admonition {
        margin-bottom: 20px;
        line-height:24px;
    }
    
    .admonition &gt; *:not(:first-child){
        /* draw a boder around a admonition */
        border-left: solid 1px #b59e9e;
        border-right: solid 1px #b59e9e;
        padding: 12px;
        margin: -12px -12px -12px -12px;
        margin-bottom: -12px !important;
    }
    .admonition &gt; .last, .admonition- &gt; .last{
        /* draw a boder around a admonition */
        border-bottom: solid 1px #b59e9e !important;
    }
    </pre></div>
    </div>
    <p>Because the standard theme set the white-space to be “no-wrap” for 
    table cell undesirable for the converted table of the GGeditor.</p>
